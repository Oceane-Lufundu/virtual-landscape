Présentation du projet

On vous demande de concevoir un GÉNÉRATEUR DE PAYSAGES VIRTUELS en java, avec l'API (application programming interface) swing.

Vous est fourni :

  • Un squelette architectural de l'application à respecter impérativement.

  • Un shell script de compilation et de génération d'un jar (build.sh).

  • Le fichier manifest qui convient pour rendre exécutable le fichier paysage.jar.

  • Un gestionnaire de positionnement (PaysageLayout.java) qui positionne automatiquement les objets de votre paysage lorsque la dimension de la fenêtre principale change.

  • Les classes

    • Paysage

      Source : Paysage.java

      Responsabilités : Générateur de paysage aléatoire.

      Remarque : Nécessitera des aménagements afin d'intégrer votre production.

    • ObjetDessin

      Source : ObjetDessin.java

      Responsabilités : Un modèle mour tout objet présent et futur du paysage.

      Remarque : Une classe abstraite qui factorise le comportement commun de tout objet dans le paysage.

    • Immeuble

      Source : Immeuble.java

      Responsabilités : Représente un "immeuble" dans le paysage (nbEtages, nbFenetres par étage, couleur paramétrables).

      Remarque : Une classe enfant de ObjetDessin.

    • Planete

      Source : Planete.java

      Responsabilités : Représente une "planète" dans le paysage.

      Remarque : Une classe enfant de ObjetDessin.

Exemple d'utilisation de l'existant :

Compilation : 

    ./build.sh

Exécution : 

    java -jar paysage.jar

Exemple de résultat :

Remarque : build.sh compile toutes les classes du projet puis construit le fichier de déploiement (paysage.jar). En phase de test, pour ne compiler qu'une seule classe, vous pouvez utiliser la commande : 

    javac Soleil.java -d .

Attention, ne pas oublier les arguments -d . afin de placer le fichier compilé (Soleil.class) au bon endroit. Il ne doit exister aucun .class dans le même répertoire que les sources (.java), si c'est le cas (mauvaise manipulation de votre part), supprimez-les.

Positionnement des objets

  • La position en x des objets est déterminée à la création de l'objet (voir la méthode init dans Paysage.java).

  • La position en y est calculée par la méthode placeToi(int horizon) définie dans la classe abstraite ObjetDessin et actionnée par le gestionnaire de positionnement PaysageLayout. Le paramètre horizon correspond à la coordonnée en y du bas de la fenêtre.

    Par défaut les objets sont au bas de la fenêtre (simulaton d'une pesanteur). Pour les décoller du bas de la fenêtre, vous donnez, à chacun, une valeur de deltaHorizon (un attribut d'objet) qui soit supérieure à zéro. Voir un exemple de cette technique au moment de la création de la planète dans Paysage.java où le dernier argument (50) correspond à la valeur de ce de deltaHorizon.

Autres caractéristiques des objets

  • Dimensions

    Elles sont déterminées aléatoirement par le générateur de paysage (méthode init de Paysage).

  • Couleur des objets

    Déterminée aléatoirement par le générateur de paysage (méthode init de Paysage).

    Attention, les objets doivent connaître la couleur de fond afin de mieux se fondre dans le paysage. Ceci est déjà pris en compte par la classe abstraite ObjetDessin et ses attributs this.couleurTrait et this.couleurFond.

  • Positionnement des objets entre eux.

    Sous la responsabilité du gestionnaire de postionnement PaysageLayout qui, en l'état, dessine en premier la (ou les) planète(s), puis les immeubles.

    Vous devrez donc intervenir sur cette classe afin qu'elle puisse prendre en compte vos créations (nouvelles classes).

Votre travail

Ce que l'on attend de vous :

  1. Un diagramme d'architecture UML de votre application dans son ensemble, sous la forme d'un diagramme de classes, à l'image du diagramme de classes du zoo et les animaux, mais en un peu plus détaillé (présenter les attributs et méthodes mais pas les accesseurs).

  2. Créer un menu pour la sauvegarde et la restauration d'un paysage.

    Puis vous ajouterez une entrée A propos qui présente votre projet (origne du projet, vos noms et prénoms, date...)

    Extra : Vous pouvez ajouter un menu "paramètres", pour le nombre de planète, coordonnées, etc.

  3. Concevoir une classe Soleil, descendant de ObjetDessin, représentant un... soleil.

    Ajouter une instance de votre classe Soleil dans le paysage.

  4. Concevoir un nouvel objet terrestre de votre choix, dont la classe est un descendant de ObjetDessin.

Vous livrerez un rapport, format HTML, composé de 3 minumum à 4 pages maximum, contenant :

  • Un document d'analyse des points qui vous ont parus sensibles, extraits de méthodes et commentaires associées (descriptif du problème et de la solution retenue), diagramme de classes UML, et copies écran de votre oeuvre.

  • Code source de votre application (sur disquette). Les nouvelles classes devront intégrer en commentaire vos noms et date de création.

Sous Linux vous pouvez utiliser Netscape Composer comme éditeur HTML, dia comme éditeur de diagramme UML et gimp pour les copies écran.

Critères d'évaluation de votre travail

Seront particulièrement appréciés

  • L'originalité et qualité de vos apports. (10 points)

  • La qualité de la documentation externe : Rapport HTML et diagramme UML. (5 points)

  • La qualité de la documentation interne : Indentation, choix des noms de variable, identificateurs, commentaires dans le code. (5 points)

Bon à savoir

Les projets feront l'objet d'un vote de la classe.

Les meilleurs projets seront publiés sur le net.
